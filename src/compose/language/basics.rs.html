5    ## üß© Variables, Let Bindings, and Scoping
6
7    Variables in Compose are introduced using the `let` keyword. Every variable has a *scope*‚Äîthe region of code where it is visible‚Äîand Compose is designed to make variable behavior clear and predictable.
8
9    ### üìå Let Bindings
10
11    To bind a value to a name, use `let`:
12
13    ```compose
14    let x = 10;
15    let name = "Compose";
16    ```
17
18    By default, variables are **immutable**‚Äîonce assigned, they cannot be changed:
19
20    ```compose error(E0004)
21    let x = 5;
22    x = 6;
23    ```
24
25    ```output error(E0004)
26    // autogenerated
27    ```
28
29    If you want to change the value later, use `let mut` to make the variable mutable:
30
31    ```compose
32    let mut count = 0;
33    count = count + 1;
34    assert::eq(count, 1);
35    ```
36
37    > Compose encourages immutability for clarity, but allows mutation when it‚Äôs explicitly requested.
38
39    ---
40
41    ### üö´ Uninitialized Bindings
42
43    In Compose, you can declare a variable without giving it an initial value.
44
45    This can be useful when multiple branches assign different values to the same variable:
46
47    ```compose
48    let x;
49    if (true) {
50        x = 1;
51    } else {
52        x = 2;
53    };
54    assert::eq(x, 1);
55    ```
56
57    > However, in this instance we might use the fact that `if` is an expression to assign the intended value to x directly.
58    > ```compose
59    > let x = if (true) { 1; } else { 2; };
60    > assert::eq(x, 1);
61    > ```
62
63    Reading an uninitialized variable is allowed and resolves to `()` (the unit value).
64    However, this produces a warning and relying on this behaviour should be avoided.
65
66    ```compose warn(W0001)
67    let x;
68    assert::eq(x, ());
69    ```
70
71    ```output warn(W0001)
72    // autogenerated
73    ```
74
75    ---
76
77    ### üìö Variable Scope
78
79    The **scope** of a variable starts where it is declared and ends at the closing brace of the block that contains it.
80
81    ```compose error(E0011)
82    let outer = 10;
83    {
84        let inner = 20;
85        assert::eq(outer, 10);
86        assert::eq(inner, 20);
87    };
88    assert::eq(inner, 10); // inner is out of scope here
89    ```
90
91    ```output error(E0011)
92    // autogenerated
93    ```
94
95    Variables can shadow earlier bindings in a new scope:
96
97    ```compose
98    let value = 1;
99    {
100        let value = value + 1;
101        assert::eq(value, 2);
102    };
103    assert::eq(value, 1);
104    ```
105
106    Variables can also be shadowed in the same scope:
107
108    ```compose
109    let x = 1;
110    let x = x + 1; // shadowing, not mutation
111    assert::eq(x, 2);
112    ```
113
114    > Shadowing is a way to ‚Äúreuse‚Äù names safely. Each binding is separate and lives in its own scope.
115
116    ---
117
118    ### ‚úÖ Summary
119
120    * Variables are immutable by default. Use `let mut` to allow mutation.
121    * Use `let` to bind values or patterns.
122    * Variables can be uninitialised and resolve to `()` before assignment.
123    * Scoping rules help prevent accidental usage of invalid or stale values.
124    * Shadowing lets you reuse names in a controlled and explicit way.
125    */

130    /**
131    ## üß† Functions and Closures
132
133    Functions and closures are essential building blocks in Compose. They allow you to reuse logic, abstract behavior, and structure your code clearly and concisely.
134
135    Let‚Äôs walk through how function calls work, how to define closures, and how variable capturing works in Compose.
136
137    ---
138
139    ### ‚òéÔ∏è Calling Functions
140
141    Function calls in Compose use parentheses to pass arguments, just like most languages:
142
143    ```compose
144    # let add = { x, y => x + y; };
145    print("Hello!");
146    add(1, 2);
147    ```
148
149    You can call:
150
151    * Built-in functions like `print`, `assert::eq`, etc.
152    * User-defined closures or functions
153
154    Function arguments are **evaluated left to right** and passed by value unless a reference is explicitly requested (e.g., via `ref` or `ref mut`).
155
156    ---
157
158    ### üß¨ Closures: Inline Functions
159
160    Closures are anonymous functions that you can assign to variables, pass around, and call later.
161
162    Basic syntax:
163
164    ```compose
165    let double = { x => x * 2; };
166    assert::eq(double(5), 10);
167    ```
168
169    Closures can take any number of arguments and produce a value:
170
171    ```compose
172    let greet = { name => "Hi, " + name + "!"; };
173    assert::eq(greet("Pieter"), "Hi, Pieter!");
174    ```
175
176    > Closures are **expressions**, so you can use them inline, assign them to variables, or return them from other functions.
177
178    ---
179
180    ### üß≤ Capturing Variables
181
182    Closures can **capture variables** from their surrounding environment. In Compose, only **boxed values** (`box`) can be captured by reference. This helps ensure clarity and memory safety.
183
184    Variables need to be captured explicitly using a capture list: `let closure = { |x| => x + 1; };` (the `|x|` part is the capture list).
185
186
187    #### ‚úÖ Capturing by Reference
188
189    ```compose
190    let mut b = box::new(10);
191
192    let get = { |ref b| => *b; };
193    assert::eq(get(), 10);
194
195    *b = 20;
196    assert::eq(get(), 20); // Closure sees the updated value
197    ```
198
199    This is powerful when closures need to access or mutate shared state.
200
201    #### üßØ Capturing Mutably
202
203    To mutate a captured box inside a closure, use `ref mut`:
204
205    ```compose
206    let mut b = box::new(1);
207
208    let inc = { |ref mut b| => *b = *b + 1; };
209
210    inc();
211    assert::eq(*b, 2);
212    ```
213
214    ---
215
216    ### üîí Owned Captures (by value)
217
218    Compose also supports *owned captures*, where the closure takes ownership of a value from its environment:
219
220    ```compose
221    let a = 42;
222    let closure = { |a| => a + 1; };
223    assert::eq(closure(), 43);
224    ```
225
226    > Owned captures cannot refer to boxed values (you must use `ref` or `ref mut` instead).
227
228    ---
229
230    ### üõ† Higher-Order Usage
231
232    Closures can be returned from functions or passed to other functions:
233
234    ```compose
235    let make_adder = { base => { |base| x => x + base; }; };
236
237    let add_five = make_adder(5);
238    assert::eq(add_five(3), 8);
239    ```
240
241    Compose‚Äôs support for closures enables expressive patterns without boilerplate.
242
243    ---
244
245    ### ‚úÖ Summary
246
247    ```markdown
248    | Feature                | Syntax                           | Notes                                   |
249    | ---------------------- | -------------------------------- | --------------------------------------- |
250    | Basic closure          | `(x) => x * 2`                   | Anonymous function                      |
251    | Call closure           | `f(5)`                           | Works just like calling a function      |
252    | Capture by reference   | `|ref y| () => *y + 1`           | Can read the captured boxed value       |
253    | Capture by mut ref     | `|ref mut y| () => { *y += 1; }` | Can mutate the captured boxed value     |
254    | Owned capture (by val) | `|y| () => y + 1`                | Captures a copy (non-boxed values only) |
255    ```
256
257    Compose makes closures easy to write and reason about, with clear scoping and safe capturing.
258    */
259    pub mod C4_Functions_and_closures {}
260);
